# Contributing

## How to contribute

1. Fork the repository.
2. Create a feature branch from `main`.
3. Make your changes.
4. Run `pnpm check` to verify your changes.
5. Open a PR against `main`.

## PR guidelines

- Title: concise, prefixed with the area (e.g. `feat(portal):`, `fix(api):`,
  `docs:`, `refactor:`).
- Description: link to the issue, summary of the change, testing notes.
- One PR = one concern. Avoid mixing refactors with features.
- Squash-merge on approval.

## Code of conduct

Be respectful. Assume good intent. Reviewers are expected to provide
constructive, actionable feedback. Contributors are expected to respond
to feedback in a timely manner.
